# Contributing

## Development workflow

1. Create a branch from the default branch.
2. Install dev dependencies with `python -m pip install -r requirements-dev.txt`.
3. Run `pytest` before opening a pull request.
4. Keep pull requests small and focused.

## Pull request requirements

- All GitHub Actions checks must pass.
- Changes should include tests when behavior changes.
- Security-sensitive changes should call out risk and mitigation in the PR description.
