# Testing And Quality

The project uses several categories of tests because no single test type catches everything.

## Unit tests

Unit tests validate the deterministic generator and symbol compatibility rules.

## Component tests

Component tests verify that the React workbench renders and responds to user interaction.

## Accessibility tests

Accessibility tests use automated auditing to catch common issues early.

## End-to-end tests

Playwright tests load the built app in a real browser and exercise the main workflow.

## Repository policy tests

Python-based repository tests verify that the project keeps its governance and GitHub automation files in place.
